实际上搞了两遍，第一遍中我的 github 账号不知道发生了什么错误，让我卡在了最后一部分
于是又注册了一个账号，有了前面的经验很快就都搞定了。由于第一遍的记忆已经模糊，便以基本顺利的第二次搭建为主，记录这段过程.

首先就是找几个好像靠谱的教程，选择其中一个照着做
下载安装 git 与 node.js 注册 github 账号
在 github 中创建仓库其中名字要符合格式
接着安装 hexo,(在第一遍中原本要在安装在 D 盘的，但是忘记改路径结果就是将其安装在 c 盘的用户文件夹...
在进行了各种搜索，比如问 ai 得到了一些指令删除其重装了)
按教程输入些指令后，在本地打开了 hexo (因为前面乱搞了一通，要安装的文件夹里有文件导致第二次安装时失败，看不懂显示的字符，截图问 ai 才知道原因)


之后是发布到 github 上
输入些指令，得到 ssh 密钥
然而当初我先是一直找不到 ssh 密钥文件夹 (因为没仔细看上面的英文没有输入路径，还以为已经生成了)
好不容易得到了密钥按照教程进行各种设置，感觉没有错误，出来的却是 404

这个过程中账号名和邮箱输了很多遍，一开始还因为在 git 上用快捷键复制出现了多余字符还没有发现多出来了字符，搞了半天
就差一点却一直搞不出来，确实让人烦躁，而且我还发现为什么我建立的仓库还不让我建分支？

最后我决定重头开始检查，并重搞了 github 账号
再搞一次我是心累的，然我意外的是接下来却很顺利，应该是有了前面的经验，我能注意到更多的细节，中间有些小问题搜索一番很快就解决了
出现了前面前面没有的 github 提示

这是几个教程里都没有的东西，但看到这个我马上明白了胜利就在眼前，接下来就好像电脑登录手机账号一样简简单单
于是有了这个网站 (嫌原界面不好看，还下载了主题，期间又是各种问题，好在应该是解决了？至少能看了)
